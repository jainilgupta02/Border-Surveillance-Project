🌿 Git Workflow - Border Surveillance AI

Complete Branch Strategy & Development Workflow

📋 Step-by-Step Workflow

🎬 Step 1: Clone Repository (First Time Only)

git clone https://github.com/jainilgupta02/Border-Surveillance-Project.git
cd Border-Surveillance-Project

🌿 Step 2: Create Your Feature Branch

git checkout develop
git pull origin develop
git checkout -b yourname/feature-description
git push -u origin yourname/feature-description

Example: git checkout -b rahul/alert-system

💻 Step 3: Work on Your Feature

Make changes to your code, test locally, and commit frequently.

# Check what changed
git status

# Add files
git add .

# Commit with meaningful message
git commit -m "feat: implement email notifications"

# Push to YOUR branch
git push origin yourname/feature-description

🔄 Step 4: Keep Your Branch Updated

Regularly sync with develop to avoid conflicts

git checkout develop
git pull origin develop
git checkout yourname/feature-description
git merge develop

📤 Step 5: Create Pull Request

When your feature is complete:

  1. Push final changes: git push origin yourname/feature-description
  2. Go to GitHub repository
  3. Click "Pull requests" → "New pull request"
  4. Base: develop | Compare: yourname/feature-description
  5. Fill in PR template with description
  6. Request review from @jainilgupta02

👀 Step 6: Code Review

Jainil will review your code and either:

  • ✅ Approve: Your code is good, will be merged
  • 💬 Request changes: Make suggested improvements and push again

✅ Step 7: After Merge

# Get latest develop with your merged code
git checkout develop
git pull origin develop

# Delete old feature branch (optional)
git branch -d yourname/feature-description

# Create new branch for next feature
git checkout -b yourname/next-feature

⚠️ CRITICAL RULES - NEVER DO THESE:

  • ❌ Never commit directly to main branch
  • ❌ Never commit directly to develop branch
  • ❌ Never merge your own Pull Request
  • ❌ Never commit .env files or API keys
  • ❌ Never push without testing locally

✅ ALWAYS DO THESE:

  • ✅ Always create a feature branch with format: yourname/feature-name
  • ✅ Always pull latest develop before starting work
  • ✅ Always write meaningful commit messages
  • ✅ Always test your code before committing
  • ✅ Always create Pull Request for review

Type Example When to Use
